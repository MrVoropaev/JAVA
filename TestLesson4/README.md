## _Ответьте письменно на вопросы:_

###  *Почему использование тестовых заглушек может быть полезным при написании модульных тестов?*
Использование тестовых заглушек может быть полезным при написании модульных тестов, так как они позволяют изолировать код, который мы хотим протестировать, от зависимостей на другие компоненты или внешние ресурсы. Тестовые заглушки позволяют нам контролировать поведение зависимых компонентов и создавать предсказуемые сценарии для тестирования.

 ### *Какой тип тестовой заглушки следует использовать, если вам нужно проверить, что метод был вызван с определенными аргументами?* 
 Для проверки, что метод был вызван с определенными аргументами, следует использовать тестовую заглушку типа "Mock". Мок-объекты позволяют нам задать ожидания для вызовов методов и проверить, что они были выполнены с нужными аргументами.

### *Какой тип тестовой заглушки следует использовать, если вам просто нужно вернуть определенное значение или исключение в ответ на вызов метода?* 
Для возвращения определенного значения или исключения в ответ на вызов метода следует использовать тестовую заглушку типа "Stub". Стаб-объекты позволяют нам задать фиксированные значения или исключения для вызовов методов и не требуют проверки вызова метода.

### *Какой тип тестовой заглушки вы бы использовали для имитации  взаимодействия с внешним API или базой данных?* 
Для имитации взаимодействия с внешним API или базой данных следует использовать тестовую заглушку типа "Mock". Мок-объекты позволяют нам задать ожидания для вызовов методов и проверить, что они были выполнены с нужными аргументами. Таким образом, мы можем имитировать ответы от внешнего API или базы данных и проверить, что наш код правильно обрабатывает эти ответы.
